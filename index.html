    DNS

    Quand je tape www.google.com dans le bar d'adresses dans mon browser, il envoie un message vers les serveurs de Google demandant le page web.

    Mon ordi a besoin de connaitre l'adresse IP du serveur de Google. Pour faire ça, il contacte un serveur de nom de domaine (DNS en anglais).

    Ce serveur de nom de domaines est un ordinateur spéciale sur l'Internet qui gardes des tables de noms de domaine et adresses IP correspondant.

    Algorithmes de recherche

    Quand tu tapes un URL dans le bar d'adresse d'un browser web, ton ordinateur consultes le serveur DNS pour connaitre l'adresse IP.

    Le serveur DNS va chercher le nom de domaine dans ses listes et trouver l'adresse IP qui correspond.

    Pour cela le serveur a besoin d'un algorithme de recherche.

    [chat,chien,cheval,souris,lapin,perroquet,éléphant]

    Chercher les noms de domaines

    Dans notre logiciel Scratch nous avons un liste de noms de domaines, et un liste d'adresses IP.

    On a besoin de mener une recherche linéaire sur la liste de noms de domaines afin de trouver le position d'un nom qu'on donne.


    Pour Commencer

    Ajouter des variables.

    1. Un qui s'appelle domain
    2. Un qui s'appelle index

    Un algorithme de recherche lineaire

    • Quand le script demarre il demande le nom d'un domaine.
      • Le variable domain est mis en réponse.
      • Le variable index est mis à 1
      • Dans un boucle qui répete autant de fois qu'il y a entrées dans le liste Domain Names.
        • Si index du liste Domain Name = le variable domain
          • Dit index du liste Adresse IP
        • Sinon
          • Incrément le variable index +1

    Pseudocode

    
quand DrapeauVert cliqué
	demande "Quel domaine cherches-tu ?"
	mets domain à réponse
	mets index à 1
	repete (longeur de Domain Names)
		si (index de Domain Names) = domain
			dit ("l'adresse IP est", index de IP Addresses)
		sinon
			changer index + 1

    Packets(Les paquets)

    Comment les ordinateurs communiquent sur un réseau

    Ce qui suit est un explication simplifié comment les ordinateurs dans un réseau passent les messages entre eux.

    1. L'ordinateur qui envoie le data a besoin de le casser en petits morceaux, qui s'appellent paquets.
    2. Chaque paquet contient (entre autre) l'adresse IP de l'ordinateur qui envoie le data, un morceau du data et l'adresse IP de l'ordinateur destinataire.
    3. L'ordinateur envoie le paquet au routeur.
    4. Le routeur examine le paquet et voit l'adresse IP de l'ordinateur pour lequel il est destiné.
    5. Le routeur envoie le paquet à son déstination, l'autre ordinateur
    6. L'ordinateur déstinaire reçoit le paquet, le mets avec les autres paquet et peux voir le data originale.

    Building the router

    The router

    It's time to script the router. The router is going to find out where the packet needs to be sent and send it in the right direction.

    1. create a list called destination_list
    2. when the router receives the broadcast "at router":
    3. add x coordinate of the destination to destination_list
    4. add y coordinate of the destination to destination_list
    5. broadcast that the packet can be sent

    Pseudocode

    
when I receive "at router"
	add x position of item last of packet_list to destination_list
	add y position of item last of packet_list to destination_list
	broadcast foward_packet

    The real Internet

    The real internet is obviously much larger.

    On the internet packets are passed from one router to another, until they eventually end up on the network of the destination computer

    Also, not all the packets will go by the same route. Different packets can travel to the same computer by going across completely different countries!
